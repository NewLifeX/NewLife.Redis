## 3. 核心模块详解

### 3.1 Redis（核心客户端）

**职责：** 配置管理、连接池、命令执行引擎、Pipeline 控制。

**关键属性：**

```csharp
// 连接配置
String Server          // 逗号分隔多地址，如 "10.0.0.1:6379,10.0.0.2:6379"
String UserName        // Redis 6.0+ ACL 用户名
String Password        // 密码
Int32  Db              // 数据库索引，默认 0
Int32  Timeout         // 读写超时，默认 3000ms
Int32  Retry           // 出错重试次数，默认 3
Int32  ShieldingTime   // 不可用节点屏蔽时间，默认 10s

// Pipeline 控制
Boolean FullPipeline   // 读操作也进入管道，默认 false
Int32 AutoPipeline     // 自动管道阈值，0=禁用

// 编码器
IPacketEncoder Encoder // 序列化策略，默认 RedisJsonEncoder

// 安全
SslProtocols SslProtocol      // TLS 协议版本
X509Certificate2 Certificate  // 客户端证书

// 可观测性
PerfCounter Counter    // 性能计数器
ITracer Tracer         // APM 链路追踪
```

**执行引擎设计：**

```
Execute<T>(key, func, write)
    │
    ├─ [管道模式] 写入或 FullPipeline 时，检查管道客户端
    │    └─ AutoPipeline: 达到阈值自动 StopPipeline + StartPipeline
    │
    ├─ [读操作] 检查管道队列，有则先 StopPipeline
    │
    └─ [正常模式] 从连接池借出 RedisClient
         │
         ├─ func(client, key) 执行命令
         ├─ 成功：Counter 统计，归还连接
         └─ 失败（非 RedisException）：
              ├─ 如果重试次数未超限
              ├─ 销毁旧连接
              ├─ 切换到下一个可用节点（_idxServer++）
              └─ 记录屏蔽时间，延迟后重试
```

### 3.2 FullRedis（增强客户端）

**职责：** 在 Redis 基础上追加集群感知、键前缀、数据结构工厂方法。

**集群初始化流程：**

```
InitCluster() [首次执行时，通过锁保证只初始化一次]
    │
    ├─ 获取 INFO 解析 redis_mode + role + connected_slaves
    │
    └─ [AutoDetect = true 时]
         ├─ mode == "cluster"      → RedisCluster (CRC16 分片路由)
         ├─ mode == "sentinel"     → RedisSentinel (哨兵监控)
         ├─ standalone + slaves>0  → RedisReplication (主从复制)
         └─ mode 为空 + role 有值  → 尝试 RedisCluster，失败则 RedisReplication (kvrocks)
```

**Execute 重载（支持集群路由）：**

```
FullRedis.Execute<T>(key, func, write)
    │
    ├─ InitCluster()
    ├─ key = GetKey(key)  // 附加键前缀
    │
    ├─ [无集群] → base.Execute()
    │
    └─ [有集群] → Cluster.SelectNode(key, write)
         │
         └─ ExecuteOnNode(key, func, write, cluster, node)
              ├─ GetPool(node) 获取该节点的连接池
              ├─ 执行命令
              ├─ 成功：cluster.ResetNode(node) 重置屏蔽
              └─ 失败：
                   ├─ ReselectNode()：处理 MOVED/ASK 重定向
                   └─ 重试或延迟等待
```

## 4. RESP 协议层

### 4.1 RESP2 协议格式

```
数据类型      前缀   示例
简单字符串     +     +OK\r\n
错误          -     -ERR message\r\n
整数          :     :1000\r\n
批量字符串     $     $6\r\nfoobar\r\n
空字符串       $     $-1\r\n
数组          *     *2\r\n$3\r\nfoo\r\n$3\r\nbar\r\n
空数组         *     *-1\r\n
```

### 4.2 RedisClient 执行流程

```
Execute<T>(command, args[])
    │
    ├─ GetStream(create=true) → 建立 TCP / TLS 连接
    │    ├─ TcpClient.Connect(addrs, port)
    │    └─ [TLS] SslStream.AuthenticateAsClient(host, cert, protocol)
    │
    ├─ [首次连接] 登录握手
    │    ├─ AUTH [username] password   // 认证
    │    ├─ SELECT db                  // 选择数据库
    │    └─ Logined = true
    │
    ├─ [管道模式] 写入管道缓冲区，延迟发送
    │
    ├─ [正常模式] 序列化命令为 RESP 格式
    │    └─ *{argc}\r\n ${len}\r\n{arg}\r\n ...
    │
    ├─ 发送字节流到 Stream
    │
    └─ 读取响应
         ├─ 读取前缀字符判断类型
         ├─ 基于类型递归解析：
         │    + → string
         │    - → 抛出 RedisException
         │    : → long → 自动转型 T
         │    $ → byte[] → 通过 IPacketEncoder 反序列化为 T
         │    * → Object[] → 递归处理每个元素
         └─ 返回 T
```

### 4.3 大缓冲区优化

- 使用 `ArrayPool<byte>` 分配读写缓冲区，减少 GC 分配
- 单次最大读取 `MAX_POOL_SIZE = 1024 * 1024`（1MB）
- Pipeline 模式下批量写入，一次 Flush

### 4.4 异步支持

`ExecuteAsync<T>` 通过 `GetStreamAsync` 建立异步连接，`ReadMoreAsync` 持续读取订阅消息，主要用于 `PubSub.SubscribeAsync`。

## 5. 连接池与并发模型

### 5.1 连接池实现

```csharp
// ObjectPool<RedisClient> 核心参数
Min = 10            // 预热连接数
Max = 100000        // 最大连接数
IdleTime = 30       // 空闲超时 30s，归还时超时则销毁
AllIdleTime = 300   // 全部空闲超时 300s，自动清空非预热连接
```

**池操作：**
- `Pool.Get()` → 借出：`Reset()` 清空残留数据 → 执行命令
- `Pool.Return(client)` → 归还
- 连接销毁时自动 `QUIT`（Logined 状态下）

### 5.2 并发安全

- `Redis` / `FullRedis` 实例本身**线程安全**，强烈建议单例复用
- `RedisClient` 单连接**非线程安全**，由连接池保障每次借出独立使用
- Pipeline 状态通过 `ThreadLocal<RedisClient?>` 实现线程隔离
- 集群初始化通过 `lock(this)` + 双重检查锁保护

### 5.3 多节点切换策略

```
_servers[]         → 地址数组（初始化时解析 Server 字符串）
_idxServer         → 当前地址索引（异常时 ++ 切换）
_nextTrace         → 切回主节点的时间戳（300s 后切回）
ShieldingTime      → 节点屏蔽窗口（默认 10s）
```

## 6. 集群与高可用架构

### 6.1 集群接口设计

```csharp
interface IRedisCluster
{
    IList<IRedisNode> Nodes { get; }
    IRedisNode? SelectNode(String key, Boolean write);
    IRedisNode? ReselectNode(String key, Boolean write, IRedisNode node, Exception ex);
    void ResetNode(IRedisNode node);
}
```

### 6.2 原生集群（RedisCluster）

```
启动时：GetNodes() → CLUSTER NODES 解析
    │
    └─ ParseNodes(nodes)
         ├─ 按 master/slave 关系构建树结构
         ├─ master 按 SLOT From 排序
         └─ 每 60s 定时刷新

SelectNode(key, write)
    ├─ 提取 key 中 {} 内容（Hash Tag）
    ├─ slot = CRC16(key) % 16384
    ├─ 筛选覆盖该 slot 的节点（LinkState==1）
    ├─ write=true → 优先 master 节点
    └─ write=false → master / slave 均可

ReselectNode(key, write, oldNode, ex)
    ├─ 解析 MOVED/ASK 错误中的新节点地址
    ├─ 找到或创建对应节点
    └─ 屏蔽旧节点
```

### 6.3 主从复制（RedisReplication）

```
GetReplications(redis, servers)
    │
    └─ 对每个地址：执行 INFO Replication
         ├─ 解析 role/connected_slaves/slave0 等字段
         ├─ 合并节点列表（主节点 + 所有从节点）
         └─ 探索式发现：从 INFO 中读到的新地址追加到探索列表

SelectNode(key, write)
    ├─ write=true → 选 master 节点
    └─ write=false → 轮询 slave 节点
```

### 6.4 哨兵（RedisSentinel）

```
StartMonitor()
    │
    └─ GetNodes() → INFO Sentinel
         ├─ 解析 master/slave 地址列表
         ├─ 探测后端是 standalone（RedisReplication）还是 cluster（RedisCluster）
         ├─ SetHostServer=true 时，更新 Redis.SetServices() 地址列表
         └─ 每 60s 定时刷新
```

## 7. 消息队列架构

### 7.1 队列类型对比

| 类型 | 底层结构 | 语义保障 | 生产复杂度 | 消费复杂度 |
|------|---------|---------|----------|----------|
| `RedisQueue<T>` | List LPUSH/BRPOP | 至多一次 | 1 次 Redis | 1 次 Redis |
| `RedisReliableQueue<T>` | List RPOPLPUSH+Ack | 至少一次 | 1 次 Redis | 2 次 Redis |
| `RedisDelayQueue<T>` | Sorted Set | 至少一次（延迟）| 1 次 Redis | 4 次 Redis |
| `RedisStream<T>` | Stream XADD/XREADGROUP/XACK | 至少一次+有序 | 1 次 Redis | 3 次 Redis |

### 7.2 可靠队列设计（RedisReliableQueue）

```
Key（主队列）               LPUSH ←── 生产者
    │
    └─ RPOPLPUSH ──→ AckKey（确认队列）  Key:Ack:{ukey}
                                           │
                                           ├─ 处理成功 → LREM AckKey 删除
                                           └─ 处理失败/崩溃 → RetryAck 周期回滚（默认 60s）
                                                │
                                                └─ 将 AckKey 中超时消息重新 RPUSH 到 Key

// 状态队列
Key:Status:{ukey}  → 每 30s 更新，证明消费者存活
                    → 全局扫描：若 Status 不活跃则回滚对应 Ack 队列
```

### 7.3 Stream 队列设计（RedisStream）

```
生产者：XADD Key MAXLEN ~ maxLength * {field:value}
                │
消费者组：       │
    SetGroup(group) → XGROUP CREATE Key group $ MKSTREAM
                │
    XREADGROUP GROUP group consumer COUNT 10 BLOCK 15000 STREAMS Key >
                │
                ├─ 处理成功 → XACK Key group msgId
                ├─ 处理失败 → 消息留在 PEL（Pending Entries List）
                │             由 RetryInterval 定时扫描死信
                │             XCLAIM 转移给当前消费者重试
                └─ 超过 MaxRetry → 丢弃

// 定时维护（DoWork, 每 10 分钟）
├─ 删除过期消息（基于 Expire 时间窗口）
└─ 自动删除空闲消费组（空闲时间 > IdleGroups 阈值）
```

## 8. 编码器与序列化

### 8.1 接口设计

```csharp
interface IPacketEncoder
{
    Object Encode(Object value);             // .NET 对象 → 可发送的格式
    T Decode<T>(IPacket data, Type type);    // 响应字节 → .NET 对象
}
```

### 8.2 RedisJsonEncoder（默认实现）

```
Encode(value):
    ├─ 基元类型（string/int/long/double/bool）→ 直接转字符串，不序列化
    └─ 复杂类型 → IJsonHost.Serialize(value) → JSON 字符串

Decode<T>(data, type):
    ├─ 基元类型 → 直接 ToXxx() 类型转换（ToInt/ToBoolean/ToDateTime等）
    └─ 复杂类型 → IJsonHost.Deserialize(json, type)
```

### 8.3 自定义编码器

实现 `IPacketEncoder` 接口即可替换：

```csharp
rds.Encoder = new MyMessagePackEncoder(); // 例如 MessagePack 二进制
```

## 11. 云厂商适配架构

### 11.1 适配原则

公有云 Redis 托管服务的共同特征：
1. **代理层**：客户端连接的是代理（Proxy），而非真实 Redis 节点
2. **认证差异**：可能要求特殊格式认证（如 `instanceId:password`）
3. **内网节点**：INFO 返回的主从 IP 是内网地址，外部不可访问
4. **SSL 要求**：部分场景强制 TLS 加密

**核心适配策略：**

## 12. 关键数据流

### 12.1 普通 Get 命令数据流

```
用户代码: rds.Get<User>("user:1")
    │
    ├─ Redis.Execute("user:1", (client, key) => client.Execute<Packet>("GET", key), write=false)
    │
    ├─ StopPipeline(check=true)  // 确保无挂起管道
    │
    ├─ Pool.Get() → 借出 RedisClient
    │
    ├─ RedisClient.Execute<Packet>("GET", "user:1")
    │    ├─ 构建 RESP：*2\r\n$3\r\nGET\r\n$6\r\nuser:1\r\n
    │    ├─ 发送到 TCP Stream
    │    └─ 读取响应 $N\r\n{json}\r\n → IPacket
    │
    ├─ Encoder.Decode<User>(packet, typeof(User))
    │    └─ JsonHost.Deserialize<User>(json)
    │
    └─ Pool.Return(client)  // 归还连接
```

### 12.2 可靠队列消费数据流

```
消费者: queue.TakeOne(timeout=30)
    │
    ├─ RetryAck()  // 检查上次未确认的消息，30s 后回滚
    │
    ├─ 执行 BRPOPLPUSH key ackKey timeout
    │    └─ 从 key 尾部弹出，原子写入 ackKey 尾部
    │
    ├─ Encoder.Decode<T>(packet, typeof(T))
    │
    ├─ 用户处理消息
    │
    └─ queue.Acknowledge(msg)  // 用户显式确认
         └─ LREM ackKey 1 {msg}  // 从确认队列删除
```

### 12.3 Stream 生产/消费数据流

```
生产者: stream.Add(new OrderMsg { ... })
    │
    └─ XADD key MAXLEN ~ 1000000 * field1 value1 field2 value2 ...
         └─ 返回 msgId（格式：timestamp-sequence）

消费者: stream.TakeOne(timeout=15)
    │
    ├─ [初次] XGROUP CREATE key group $ MKSTREAM  // 创建消费组
    │
    ├─ XREADGROUP GROUP group consumer COUNT 1 BLOCK 15000 STREAMS key >
    │    └─ 返回 [[key, [[msgId, [fields]]]]]
    │
    ├─ 解析字段数组 → 重建 T 对象
    │
    ├─ 用户处理消息
    │
    └─ stream.Acknowledge(msg.Id)
         └─ XACK key group msgId
```
